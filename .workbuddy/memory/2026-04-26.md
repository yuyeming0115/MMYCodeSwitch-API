## 图标修复

- **问题**：打包后的 `.app` 显示默认「Code」图标，而非用户自定义的彩色圆环图标
- **根因**：`src-tauri/icons/icon.png`（Tauri 主源图标）和 `icon_square.png` 存的是错误的「Code」图标，彩色圆环只存在于部分尺寸文件（如 `128x128@2x.png`）
- **修复**：
  1. 用 Python Pillow 将彩色圆环图标转为正方形（居中+透明填充）并放大到 1024×1024
  2. 覆盖 `icon.png` 和 `icon_square.png`
  3. 运行 `npx tauri icon` 重新生成所有平台图标（ICNS/ICO/PNG/iOS/Android 全套）
  4. 重新 `bash ./build.sh --portable` 打包
- **经验**：Tauri 的 `tauri icon` 要求**正方形**源图，非正方形的 PNG 会报错 `Source image must be square`
- **结果**：`.app` 内 `icon.icns` 已更新为彩色圆环图标（1.2MB），Finder 缓存已清除

## Mac 平台 3 个 Bug 修复

用户反馈 Mac 使用时的 3 个问题，均已定位根因并修复：

### 1. 标题栏无法拖拽
- **文件**: `src/components/AppContent.vue`
- **根因**: Tauri v2 在 macOS `transparent: true` 模式下，仅用 CSS `-webkit-app-region: drag` 不够
- **修复**: 标题栏 div 添加 `data-tauri-drag-region` 属性（Tauri v2 推荐方式）

### 2. 内置模板 SVG 图标未应用到供应商
- **文件**: `src/components/ProviderForm.vue`
- **根因**: `selectTemplate()` 选择模板时未记录 `builtinIcon`（如 `icons/dashscope.svg`），`submit()` 保存时 icon_path 始终为 null
- **修复**: 
  - 新增 `selectedBuiltinIcon` ref 记录选中模板的内置图标
  - `selectTemplate()` 中同步设置 `selectedBuiltinIcon` 和 `iconPreview`
  - `submit()` 中优先级：用户上传 > 模板内置图标 > 原有图标
  - `resetForm()` 中清理 `selectedBuiltinIcon`

### 3. 终端打开后未运行 Claude Code
- **文件**: `src-tauri/src/lib.rs`
- **根因**: `open -a Terminal --args claude` 只能打开 Terminal.app，但不会在其中执行 claude 命令
- **修复**: 改用 `osascript -e 'tell application "Terminal" to do script "cd {path} && claude"'` 通过 AppleScript 让 Terminal 执行命令
- **验证**: `cargo check` 编译通过，`vue-tsc` 类型检查通过

## 标题栏拖拽增强 + CLAUDE.md 标记块注入功能

### 标题栏拖拽（第二轮修复）
- **问题**：第一轮加了 `data-tauri-drag-region` 和 `startDragging()` 后用户反馈仍然无效
- **根因**：缺少 Tauri v2 capabilities 权限声明 `core:window:allow-start-dragging`
- **修复**：
  1. `src-tauri/capabilities/default.json` 添加 `core:window:allow-start-dragging`
  2. `AppContent.vue` 新增 `startWindowDrag()` 函数绑定到 `.app` 容器的 `@mousedown`
  3. 通过 `closest()` 过滤按钮/输入框/卡片等交互元素，避免误触
- **经验**：Tauri v2 的 window API 权限缺失会被静默 drop，不报错不抛异常

### 标题栏滚动消失 bug
- **问题**：内容区有较多项目时向下滚动，标题栏会消失
- **根因**：`.page-main` 设置了 `height: 100vh` 与 `.app` 的 `100vh` 叠加导致总高度超出视口
- **修复**：
  - `src/App.vue` 全局 `.app` 添加 `overflow: hidden`
  - `src/components/AppContent.vue` scoped `.app` 添加 `overflow: hidden`
  - `.page-main` 从 `height: 100vh` 改为 `flex: 1; min-height: 0`

### CLAUDE.md 标记块注入功能
- **需求**：Claude Code CLI 硬编码自称 "Claude Opus X.Y"，与实际使用模型不符
- **方案**：安全标记块注入法（不影响用户已有 CLAUDE.md 内容）
- **实现文件**：
  - `src-tauri/src/inject.rs`：新增 `inject_claude_md()` / `remove_claude_md_block()` / `build_claude_md_content()`
  - `src-tauri/src/lib.rs`：`remove_active_project()` 自动清理标记块；新增独立命令 `clean_claude_md_block`
- **标记格式**：`<!-- MMY-INJECT:START -->` ... `<!-- MMY-INJECT:END -->`
- **注入时机**：`inject_to_project_dir()` 成功写入 settings.json 后自动调用
- **清理时机**：项目解绑时（`remove_active_project`）自动移除标记块
- **安全性**：
  - 文件不存在 → 新建并只写标记块
  - 文件存在无标记块 → 追加到末尾
  - 文件存在有标记块 → 只替换标记块内部内容
  - 清理后只剩空白 → 删除整个文件
- **验证**：`cargo check` + `vue-tsc` 均通过
