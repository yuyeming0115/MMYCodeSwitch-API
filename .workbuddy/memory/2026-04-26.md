# 2026-04-26 工作日志

## Mac 便携版打包修复

- **问题**：`build.sh` 在 macOS zsh 终端下报 `bad substitution`（`${MODE^^}` 语法不兼容）和 `permission denied`
- **修复**：
  1. 将 `${MODE^^}` 替换为 POSIX 兼容的 `tr '[:lower:]' '[:upper:]'` 函数
  2. Mac 便携模式从 `--no-bundle` 改为完整构建，确保生成完整 `.app` bundle
  3. 更新 README.md，补充 macOS 打包注意事项和常见问题 FAQ
- **结果**：`bash ./build.sh --portable` 可正常输出 `MMYCodeSwitch-API-Portable/MMYCodeSwitch-API.app`（19MB）

## 图标修复

- **问题**：打包后的 `.app` 显示默认「Code」图标，而非用户自定义的彩色圆环图标
- **根因**：`src-tauri/icons/icon.png`（Tauri 主源图标）和 `icon_square.png` 存的是错误的「Code」图标，彩色圆环只存在于部分尺寸文件（如 `128x128@2x.png`）
- **修复**：
  1. 用 Python Pillow 将彩色圆环图标转为正方形（居中+透明填充）并放大到 1024×1024
  2. 覆盖 `icon.png` 和 `icon_square.png`
  3. 运行 `npx tauri icon` 重新生成所有平台图标（ICNS/ICO/PNG/iOS/Android 全套）
  4. 重新 `bash ./build.sh --portable` 打包
- **经验**：Tauri 的 `tauri icon` 要求**正方形**源图，非正方形的 PNG 会报错 `Source image must be square`
- **结果**：`.app` 内 `icon.icns` 已更新为彩色圆环图标（1.2MB），Finder 缓存已清除
