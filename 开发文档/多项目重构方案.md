## Phase 1：基础架构（已完成）

### Product Overview

MMYCodeSwitch-API 是一款 Tauri 桌面应用，用于管理多个 AI API 供应商配置并注入到 Claude Code 的 settings.json 中。当前版本只支持**单实例模式**（固定写入 `~/.claude/settings.json`），每次点击不同 Provider 都会覆盖同一个配置文件。本次改造的核心目标是实现**多项目管理模式**：用户点击某个 API 图标时，弹出一个文件夹选择框，选择一个项目目录，将 API 配置注入到该项目目录下的 `.claude/settings.json`，从而实现每个项目使用不同的 API 供应商，互不干扰。

### Core Features (Phase 1 - 已完成)

- **动态项目文件夹选择**：点击 Provider 图标后，弹出系统文件夹选择对话框，让用户选择目标项目路径，而非使用固定的 `~/.claude` 目录
- **项目级配置隔离**：每个选中的项目目录下自动创建/更新 `.claude/settings.json`，各项目的 API 配置完全独立
- **已激活项目管理面板**：在主界面展示所有已绑定 API 的项目列表卡片（显示项目名称、使用的 Provider、绑定时间），支持移除项目绑定
- **重复项目检测与确认**：当用户选择的文件夹已被其他项目绑定时，弹出确认提示「该项目已在用 XX API，是否切换？」
- **保留全局默认实例**：设置页面中可配置一个「全局默认目录」作为 fallback（默认仍为 `~/.claude`），用于未指定项目的场景

### Tech Stack

- **前端框架**：Vue 3 + TypeScript + Pinia（已有，沿用）
- **UI 组件库**：Naive UI（已有，沿用）
- **桌面框架**：Tauri 2.x + Rust（已有，沿用）
- **文件对话框**：`@tauri-apps/plugin-dialog`（已安装，用于文件夹选择）

#### 数据模型变更

**新增 ActiveProject 数据结构**：

```typescript
interface ActiveProject {
  id: string               // 唯一 ID，如 "proj_1745xxx"
  name: string             // 项目名称（取自文件夹名或用户编辑）
  project_path: string     // 项目根目录绝对路径，如 "D:/GitWork/my-project"
  provider_id: string      // 当前绑定到的 Provider ID
  provider_name: string    // 供应商显示名称
  created_at: string       // 绑定时间 ISO
  updated_at: string       // 最后更新时间 ISO
}
```

**AppConfig 变更**：

```typescript
interface AppConfig {
  language: string
  instances: Instance[]                    // legacy，保留兼容
  default_config_dir?: string              // 全局默认目录
  active_projects: ActiveProject[]         // 已激活的项目列表（核心数据）
}
```

#### 核心数据流

1. 用户点击 Provider 图标 → `ProviderGrid emit('switch', p)`
2. `AppContent.doSwitch(p)` → 调用 `open({ directory: true })` 弹出文件夹选择框
3. 用户选中目录 → 检查路径是否已存在于 `active_projects`
4. 调用 Rust command `inject_to_project(project_path, provider_id)` → 写入 settings.json + 更新记录
5. 前端刷新项目列表

## Phase 2：Bug修复 + CLI自动启动（🚧 进行中）

### 问题背景

Phase 1 完成后，用户测试发现：
1. ⚠️ **点击供应商选择文件夹后完全无反应** — 无 toast 提示、项目列表不更新、就像没点一样
2. ❌ **没有自动弹出 Claude Code CLI 终端** — 对比 JCode 的体验：选完供应商后应自动打开终端运行 claude

### Bug 根因分析

通过代码审查定位以下潜在问题点：

**问题 A：dialog open() 返回值类型**
Tauri v2 的 `@tauri-apps/plugin-dialog` 的 `open()` 在 `multiple: false` 时可能返回 `string[]`（单元素数组）而非裸 `string`。如果实际拿到的是 `["D:\\GitWork\\my-project"]`，后续 `normalizePath()` 处理后会变成错误的路径，导致：
- 路径匹配失败 → 不会检测到重复（正常路径）
- 但 inject 可能仍然执行成功（Rust 侧收到数组 toString 后的逗号拼接字符串会失败或写入错误位置）

**问题 B：异常被静默吞掉**
`doInject()` 中 `store.injectToProject()` 如果抛异常，catch 块虽然调用 `msg.error()`，但如果：
- Naive UI Message 实例未正确初始化
- 或者错误对象不是标准 Error 类型
→ 用户看不到任何提示

**问题 C：缺少诊断日志**
全链路无任何 console.log/console.error，出了问题无法在 DevTools 排查。

### 修复方案

#### Task 1: 修复注入无反应 Bug

**文件**: `src/components/AppContent.vue`, `src/stores/app.ts`

改动点：
1. **处理 dialog 返回值** — 兼容 `string | string[]` 类型
2. **添加全链路 console 日志** — doSwitch 入口、选中的路径、inject 开始/结果
3. **添加 injecting loading 状态** — 防止重复点击，给用户视觉反馈
4. **确保 msg.success/error 正确渲染**

#### Task 2: 新增 launch_terminal Rust Command

**文件**: `src-tauri/src/lib.rs`

新增 Tauri command：
```rust
#[tauri::command]
fn launch_terminal(workdir: String) -> Result<(), String>
```

实现策略（Windows 平台）：
- 使用 `std::process::Command` 启动 PowerShell / Windows Terminal
- 参数：`-NoExit -Command "claude"` 
- 工作目录设为用户选择的项目路径
- 使用 `CREATE_NEW_CONSOLE` flag 创建独立终端窗口
- 非阻塞 spawn，终端窗口脱离父进程独立运行

#### Task 3: Shell 权限配置

**文件**: `src-tauri/capabilities/default.json`

添加权限：
```json
{
  "permissions": [
    "...existing...",
    "shell:allow-execute"
  ]
}
```

#### Task 4: 集成 CLI 启动到注入流程

**文件**: `src/components/AppContent.vue`, `src/stores/app.ts`

修改 `doInject()` 流程：
```
doInject(path, p):
  1. console.log('[inject] 开始注入:', p.name, '→', path)
  2. injecting.value = true  (loading UI)
  3. result = store.injectToProject(path, p.id)
  4. ✅ msg.success('已将 XXX 注入到 YYY')
  5. 🚀 invoke('launch_terminal', { workDir: path })
  6.     msg.info('正在启动 Claude Code CLI...')
  7. injecting.value = false
```

#### Task 5: 补充 i18n 文案

**文件**: `src/i18n/zh.ts`, `src/i18n/en.ts`

新增翻译 key：
| Key | zh | en |
|-----|----|----|
| `injecting` | 正在注入配置... | Injecting config... |
| `launching_cli` | 正在启动 Claude Code CLI... | Launching Claude Code CLI... |
| `launch_failed` | 启动终端失败 | Failed to launch terminal |
